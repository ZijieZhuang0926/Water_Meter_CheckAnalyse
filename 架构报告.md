# 云拍器水表异常检测系统 - 架构报告

## 一、系统概述

本系统用于检测云拍器上报的水表读数异常，采用**规则优先 + 统计辅助**的分层架构，解决冷启动、季节/节假日影响、字轮错位/识别失败、一日多次识别等实际问题。

**核心设计原则**：
- 硬规则永远执行，确保基本物理约束不被违反
- ML预测渐进式启用（60天后），利用历史数据提升检测精度
- 可信基线机制，异常跳变后自动回退到正确基准

## 三、核心算法详解

### 3.1 可信基线查找 (_find_trusted_baseline)

**目的**：当历史数据中存在OCR识别错误时，跳过异常读数，找到最近的可信读数作为对比基准。

**多阶段检测流程**：

| 阶段 | 检测内容 | 判定标准 |
|------|---------|---------|
| 1 | 负增量检测 | `increment < 0` → 标记异常 |
| 2 | 字轮异常检测 | 基于总步数推断期望值，不匹配 → 标记异常 |
| 3 | 增量异常检测 | `increment > median + 5×MAD` → 标记异常 |
| 4 | 读数偏差异常 | 按位数分组，`\|reading - group_median\| > 30×MAD` → 标记异常 |
| 5 | 位数异常检测 | 正常读数位数众数占比>50%，不符且非合法进位 → 标记异常 |

**关键设计**：
- 按位数分组计算统计量，避免异常值污染正常组
- 允许合法进位（如...999→1000），但需验证增量足够
- 从最新数据向前查找，优先使用最近的可信读数

### 3.2 硬规则检测 (HardRuleChecker)

**检测规则列表**：

| 规则 | 代码位置 | 触发条件 | 置信度 |
|------|---------|---------|--------|
| 数据不足 | hard_rules.py#L46-47 | `len(today_readings) == 0` | 1.0 |
| 同日不一致 | hard_rules.py#L51-54 | `CV = std/mean > 0.1` | 0.8 |
| 首次上报 | hard_rules.py#L56-57 | `len(history) == 0` | 0.0（通过） |
| 位数突变+2 | hard_rules.py#L147-149 | `digit_diff >= 2` | 0.95 |
| 位数突变+1 | hard_rules.py#L151-156 | `digit_diff == 1`且`inc < min_inc` | 0.95 |
| 位数减少 | hard_rules.py#L160-162 | `digit_diff <= -1` | 0.95 |
| 读数回退 | hard_rules.py#L76-78 | `today < prev` | 1.0 |
| 日期间隔 | hard_rules.py#L80-83 | `gap > 7` | 0.9 |
| 物理上限 | hard_rules.py#L92-96 | `total_inc > 1500 * gap` | 0.95 |
| 零增量 | hard_rules.py#L98-105 | `inc <= 0.01` | 0.8 |
| 连续零增量 | hard_rules.py#L100-103 | 连续≥3天零增量 | 0.95 |
| 字轮进位 | hard_rules.py#L184-253 | 实际值≠期望值 | 0.3~0.95 |

**字轮进位检测原理**：

基于总步数（today - yesterday）从个位开始逐位进位，推断每一位的期望值，与实际值对比。

示例：昨天24648，今天24720，总步数72
- 个位：8 + 72 = 80 → 期望0，进位8
- 十位：4 + 8 = 12 → 期望2，进位1
- 百位：6 + 1 = 7 → 期望7，无进位

### 3.3 ML预测检测 (ConsumptionPredictor)

**整数数据优化策略**：

| 问题 | 解决方案 | 代码位置 |
|------|---------|---------|
| F-ratio失效 | 众数一致性比率检测周期性 | consumption_predictor.py#L161-188 |
| MAD=0 | 设置MAD下限为0.5 | consumption_predictor.py#L116 |
| OCR污染 | MAD预过滤，剔除>5×MAD的异常点 | consumption_predictor.py#L145-159 |
| 线性外推不合理 | EMA指数移动平均替代 | consumption_predictor.py#L210-239 |
| 比较空间不一致 | 预测区间±0.5整数量化补偿 | consumption_predictor.py#L121-122 |

**预测因子权重**：

| 因子 | 基础权重 | 说明 |
|------|---------|------|
| 同星期基线 | 0.6 × 周期性得分 | 周期性得分0~1，由众数一致性比率决定 |
| EMA趋势 | 0.3 | 最近14天指数移动平均 |
| 节假日调整 | 0.1 | 基于历史节假日与工作日用量比值 |

## 四、参数配置说明

### 4.1 检测器参数 (WaterMeterAnomalyDetector)

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `mad_factor` | 3.0 | 预测区间宽度倍数（预测值±factor×MAD） |
| `min_samples` | 8 | 同星期预测最小样本数 |
| `min_history_days` | 60 | 启用ML预测所需最小历史天数 |
| `weekday_weight` | 0.6 | 同星期基线权重 |
| `trend_weight` | 0.3 | EMA趋势权重 |
| `holiday_weight` | 0.1 | 节假日调整权重 |

### 4.2 硬规则参数 (HardRuleChecker)

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `physical_max_daily` | 1500 | 单日最大增量（m³） |
| `zero_threshold` | 0.01 | 零增量判定阈值 |
| `consecutive_zero_days` | 3 | 连续零增量报警天数 |
| `same_day_cv_threshold` | 0.1 | 同日读数不一致CV阈值 |
| `max_date_gap_days` | 7 | 日期间隔异常阈值 |

### 4.3 预测器参数 (ConsumptionPredictor)

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `mad_floor` | 0.5 | MAD最小值，防止整数数据MAD=0 |
| `ema_alpha` | 0.3 | EMA平滑系数 |
| `outlier_factor` | 5.0 | 异常值过滤倍数（MAD） |

## 七、已知限制与后续优化

### 7.1 当前限制

| 限制 | 说明 | 影响 |
|------|------|------|
| 冷启动阶段无增量检测 | 0-59天仅硬规则，增量异常无法检测 | 可能漏检增量异常 |
| 物理上限固定 | `physical_max_daily = 1500`不区分水表规格 | 对大口径水表可能过于宽松 |
| 节假日依赖第三方库 | `chinese_calendar`可能无法识别调休变更 | 预测精度受影响 |
| 无换表检测 | 无法区分换表（读数大幅下降）与OCR错误 | 换表需手动干预 |

### 7.2 后续优化方向

1. **增量自适应阈值**：基于历史数据动态调整增量异常阈值
2. **换表检测**：结合安装维护记录自动识别换表场景
3. **设备画像**：按设备类型/位置/用水量建立个性化检测模型
4. **异常学习**：收集误判案例，持续优化字轮检测逻辑
5. **实时检测**：支持流式数据实时异常检测
